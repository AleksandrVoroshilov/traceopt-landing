# Traceopt — Project Context

## Что это за продукт

**Traceopt** — сервис **оптимальной трассировки тепловых спутников (heat tracing)** для технологических трубопроводов промышленных установок.

Это **не CAD-плагин** и не очередной софт. Это **Engineering as a Service** от команды опытных piping-инженеров и специалистов по оптимизации.

## Почему это нужно клиентам

В крупных промышленных проектах:
- До **40 %** трубопроводов требуют обогрева тепловыми спутниками.
- 10–15 % рабочего времени piping-дизайнера уходит на проектирование heat tracing.
- Около **80 %** этих трудозатрат приходится именно на ручную проработку трасс в 3D-модели.

Поиск оптимальной трассировки — это **NP-полная задача**.

## Что даёт Traceopt

- **Скорость**: от дней/недель/месяцев → минуты
- **Оптимальность**: существенно короче трассы, минимум материалов и fittings
- **Качество**: детерминистическое + метаэвристическое решение (RPS-ER + ALNS)

## Как работает

**Вход**: PCF-файлы или экспорт 3D-модели трубопроводов  
**Выход**: оптимальные маршруты tracers в формате (x, y, z) полилиний + BOM

## Архитектура (высокий уровень)

1. Import → JSON
2. Bridge detection
3. Graph contraction
4. Основной солвер **RPS-ER** (arc-routing)
5. Улучшение через **ALNS** (Adaptive Large Neighborhood Search)
6. Export (полилинии + отчёты)

## Цель лендинга

Создать доверие у senior piping-инженеров, EPC-контракторов и проектных институтов, чётко показать:
- Глубокое понимание боли
- Профессионализм команды
- Значимый инженерный результат (а не «ещё один красивый инструмент»)
